Quick Start.

Sign up and run your first SLM query in under 5 minutes. Works with any OpenAI-compatible client.

01Sign in 02Pick a client 03First query

01 · Sign in

Get your API key

Go to slm.dev/sign-in and authorize with GitHub or Google. Your API key is generated automatically and visible on the dashboard. Keys start with slm_ followed by a 32-char hex token.

HEADS UP
Anonymous use works too — you get 5 req/min and 10K tokens/day with no signup. Sign in to lift those limits.

03 · First query

Hit the API with curl

BASHcopy
curl https://api.slm.dev/v1/chat/completions \
  -H "Authorization: Bearer slm_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "slm-solana",
    "messages": [{"role": "user", "content": "How do I derive a PDA in Anchor?"}],
    "max_tokens": 512
  }'

The endpoint is a drop-in replacement for OpenAI's /v1/chat/completions. Pass "stream": true for SSE. Rate limits depend on your tier.

04 Guardrails Why the model doesn't hallucinate outdated patterns
NO · DECLARE_ID

Never emits declare_id!. Always uses declare_program! per Anchor 0.30+.

NO · REENTRANCY

Doesn't suggest reentrancy guards — Solana prevents reentrancy via CPI depth limits.

NO · CORAL-XYZ

References solana-foundation/anchor, not the old coral-xyz org.

NO · CLOSED DISCRIMINATOR

No warnings about closed-account discriminator attacks — patched years ago.

NO · FLOAT CONCERN

Doesn't warn about float non-determinism. Solana compute is deterministic.

YES · MODERN IX

Uses get_instruction_relative, not the deprecated load_instruction_at.

Plain english streaming

This transaction is an SPL token swap. The wallet 7xKXT…8Qmn swapped 2.5 SOL for ~428.91 USDC through the Jupiter aggregator, which routed through Raydium CLMM.

What happened:

  1. Associated Token Account for USDC was created for the signer (one-time setup, rent: 0.00203 SOL).
  2. 2.5 SOL was wrapped to wSOL via the Token program.
  3. Jupiter routed the swap through Raydium CLMM pool SOL/USDC 0.05%.
  4. 428.91 USDC was transferred to the signer's ATA; the wSOL account was closed.

Slippage realized: 0.12%. No errors; all 4 instructions succeeded in a single atomic transaction.
